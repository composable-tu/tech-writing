# Technical Writing Checklist

Use this checklist when writing or reviewing a technical text.

## General Style

- [ ] Narrative is relaxed and professional
- [ ] No flirtation, memes, or jokes
- [ ] Verbal nouns replaced with verbs
- [ ] Passive voice minimized or eliminated
- [ ] Imperative mood used for instructions

## Structure

- [ ] Text is divided into paragraphs
- [ ] Each paragraph contains one main thought
- [ ] Paragraphs begin with the main idea
- [ ] Sentences are short
- [ ] Syntax is simple (no excessive commas, participles, brackets)
- [ ] Text is read aloud and flows smoothly
- [ ] Headings and lists are used for modularity
- [ ] Introduction explains usefulness and answers:
  - [ ] What concept will be introduced?
  - [ ] What will the reader be able to do after reading?
  - [ ] Why is it necessary?
- [ ] Conclusion summarizes and gives practical advice
- [ ] Conclusion includes references for further study

## Editing

- [ ] Terminology and abbreviations are minimized
- [ ] Complex terms are explained
- [ ] Cause and effect are explained
- [ ] Introduction is short
- [ ] No filler phrases like "pay attention" or "it is worth mentioning"
- [ ] Text is as short as possible without losing meaning
- [ ] No unnecessary introductory words, participles, or synonyms
- [ ] Claims are objective and supported by facts
- [ ] Language is unambiguous
- [ ] Subjective evaluations are removed or backed by evidence
- [ ] Diagrams, screenshots, or code samples are used where helpful

## Code Examples

- [ ] No `foo` or `bar`
- [ ] Variable names are expressive and self-documenting
- [ ] Nouns for objects/primitives, plural nouns for arrays
- [ ] Function names begin with verbs
- [ ] Event handlers use `on` or `handle` prefix
- [ ] Booleans use `is` or `has` prefix
- [ ] No magic numbers
- [ ] Imports shown when code from a module is first used
- [ ] Examples start simple and gradually increase complexity
- [ ] Complex code has comments and additional explanation
- [ ] Terminal commands include expected output
- [ ] Anti-patterns are highlighted if relevant
- [ ] Code samples are verified to work
- [ ] Content is up to date with current technology

## Final Steps

- [ ] Text edited after first draft
- [ ] Text read aloud
- [ ] Feedback requested from colleagues or peers
- [ ] Text carefully proofread before publishing
